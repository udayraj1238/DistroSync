01 system design

Three roles. One protocol.

Every component speaks the same 4-byte length-prefixed JSON wire format over raw TCP. No message broker library, no ORM, no framework — the protocol is the product.

03 under the hood

Six mechanisms. Built, not borrowed.

Every one of these exists because a production message queue needs it — not because it looked good in a demo.

01
asyncio event loop
A single thread handles thousands of concurrent TCP connections without blocking, using cooperative multitasking instead of one-thread-per-connection.
broker/server.py
02
ProcessPoolExecutor
Python's GIL blocks true CPU parallelism with threads. Workers bypass it entirely by running tasks in separate OS processes.
worker/task_executor.py
03
Adaptive token bucket
Fill rate adjusts live to queue depth and worker latency. Under 5× overload it throttles to 5% of normal — self-regulation, not a crash.
broker/load_shedder.py
04
SQLite WAL mode
Every task state change is appended to a write-ahead log before touching the database. Zero tasks lost across 10 simulated crashes.
persistence/wal_store.py
05
Heartbeat eviction
Workers ping every 2 seconds. Three missed beats evicts the worker and re-queues its in-flight tasks — the same pattern Kubernetes uses.
broker/worker_registry.py
06
Dead letter queue + replay
Tasks failing 3 times move to the DLQ instead of vanishing. An operator inspects, fixes the bug, and replays — exactly how SQS and RabbitMQ work.
broker/dead_letter.py
